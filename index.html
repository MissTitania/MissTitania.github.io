This is an in-browser AI implementation for the classic game Othello. If you haven't played before, I recommend you start here with the rules.

Neural nets were implemented and trained in java. This AI runs a minimax search with alpha-beta pruning heuristics. There's no transposition table. The collision rate in Othello is too low to be worthwhile. A pre-computed function evaluates mid-game board states. The differential of legal moves is one feature. Another feature is the perfect-play results of sub-games of different board subsets.

The technique stores boards as 2 64-bit integers (1 for each player) marking all piece positions with a 1 bit. This allows very fast computations with bitboard manipulations and pre-computed masks.

The server of Pokemon Showdown (a community for competitively-minded Pokemon battling as a strategy game) called Azure desperately needed an interface remodeling. This was for reasons of readability, aesthetics, and ease of use. They hired me to remake their CSS for this purpose, but the catch was that esoteric rules in their license agreement with the larger Showdown community allow them only to modify server-side code. They must use the same client as the rest of Pokemon Showdown verbatim, differing only by a style sheet modification. So my task was to update to a more modern interface without modifying either the underlying JS code or the DOM. The result contained a great deal of animation, both for a sleeker-looking product and intuitive identification of what things on screen were even control elements. This means that for an authentic experience I would encourage you to try injecting the style sheets yourself and see it all in action! Still pictures can only tell so much sadly

For starters, PS is a battle-based entity, so let's cut right to the chase with the battle interface:

Moving the team and name display sidebars up top creates a lot more space to show off all the actual battle! The graphical space in the center is where all the attack, terrain, and weather animations play out, so playing that up easily improves the user experience. But the biggest gain of all isn't even visible in the picture. See that little notch at the bottom of the re-designed battlefield? As moves are made, all of them get called out by trainers in speech bubbles right there. Beforehand all these actions merely got dumped in a chat log elsewhere. Surely one of the best ways to ruin a child's fun Pokemon experience is forcing them to scroll back through textual data dumps to see what's happening, so the change was very much overdo. As you can see, even short conversations few in number quickly overwhelm the space allocated for them. That main menu on the left doesn't do a whole lot except for when you're queueing up or configuring your team for battle; it just eats up direct message space.
